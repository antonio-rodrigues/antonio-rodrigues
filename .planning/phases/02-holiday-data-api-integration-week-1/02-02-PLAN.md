    Create `src/composables/useHolidays.ts` implementing the full composable:

    ```
    Export: HolidayEntry interface, useHolidays function
    ```

    **HolidayEntry interface:**
    ```typescript
    export interface HolidayEntry {
      name: string
      type: 'national' | 'municipal'
      municipalityName?: string
    }
    ```

    **useHolidays() function:**
    - Uses `useConfigStore()` for year and selectedMunicipalityId
    - `loading: Ref<boolean>`, `error: Ref<string | null>`
    - `nationalHolidays: Ref<Map<string, HolidayEntry>>` initialized as `ref(new Map())`
    - `fetchHolidays(year)`: fetch from `https://date.nager.at/api/v3/PublicHolidays/${year}/PT`, filter `h.global`, build Map. On error → call `buildFallbackHolidays(year)`. Always replace ref (never mutate in place — avoids Vue reactivity pitfall).
    - `buildFallbackHolidays(year)`: Use functions from `holiday-utils.ts`. Build a Map with the fixed national holidays Portugal has year-round. Use `format(date, 'yyyy-MM-dd')` from date-fns for keys.
    - `resolveMunicipalHolidayDate(holiday, year)`: if `holiday.type === 'fixed'` → `format(new Date(year, holiday.month - 1, holiday.day), 'yyyy-MM-dd')`. If `mobile` → `format(addDays(getEaster(year), holiday.offset!), 'yyyy-MM-dd')`.
    - `municipalHoliday: ComputedRef` — watches `configStore.selectedMunicipalityId`. Finds municipality in municipalitiesData, resolves date, returns `{dateStr, entry}` or null.
    - `holidays: ComputedRef<Map<string, HolidayEntry>>` — merges nationalHolidays + municipalHoliday. Create `new Map(nationalHolidays.value)` then `.set()` municipal entry if present.
    - `watch(() => configStore.year, (year) => fetchHolidays(year), { immediate: true })`
    - Returns: `{ holidays, loading, error }`

    Import `municipalitiesData` from `'../data/municipalities.json'`. Add `assert { type: 'json' }` only if TypeScript requires it — check tsconfig first; Vite handles JSON imports natively without it.

    Anti-patterns to avoid (from research):
    - Never call useHolidays() inside MonthCard or DayCell
    - Never mutate the Map in-place — always replace the ref value
    - Never fetch per-month — one fetch per year
