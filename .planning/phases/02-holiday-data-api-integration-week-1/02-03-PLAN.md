    **State:**
    - `query: Ref<string>` — bound to the input with v-model
    - `isOpen: Ref<boolean>` — controls dropdown visibility
    - `configStore = useConfigStore()`

    **Computed:**
    ```typescript
    const filtered = computed(() =>
      query.value.length < 2
        ? []
        : municipalitiesData
            .filter(m => m.name.toLowerCase().includes(query.value.toLowerCase()))
            .slice(0, 10)
    )
    ```

    **Methods:**
    - `selectMunicipality(mun)`: set `configStore.selectedMunicipalityId = mun.id`, set `query = mun.name`, close dropdown
    - `clearSelection()`: watch the query — if it becomes empty, set `configStore.selectedMunicipalityId = null`
    - Handle clear: use `watch(query, val => { if (!val) configStore.selectedMunicipalityId = null })`

    **Template:**
    - `<div class="relative">` as wrapper
    - Text input with placeholder "Pesquisar município..." bound to `query`
    - Dropdown list (`v-if="filtered.length > 0"`) below the input — position absolute, white bg, shadow, border, rounded
    - Each list item: `<li @click="selectMunicipality(mun)">` showing `mun.name` + `<span class="text-gray-400 text-xs">{{ mun.district }}</span>`
    - Dismiss dropdown on `@blur` of the input with a 150ms delay (`setTimeout`) to allow click to register first

    Tailwind classes to use (per project conventions):
    - Input: `w-full border border-gray-300 rounded-md px-3 py-2 text-sm focus:outline-none focus:ring-2 focus:ring-blue-500`
    - Dropdown: `absolute z-10 mt-1 w-full bg-white border border-gray-200 rounded-md shadow-lg max-h-60 overflow-auto`
    - List item: `px-3 py-2 text-sm cursor-pointer hover:bg-gray-100`
